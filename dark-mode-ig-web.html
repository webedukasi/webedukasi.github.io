Jawaban:

Deep Web

Deep Web adalah sekumpulan website yang mana tidak terindex oleh mesin pencari seperti Bing, Yahoo dan Google. Jumlah website yang ada di Deep Web jauh melebihi website-website yang terindex oleh mesin pencari konvensional. Untuk menjelajah di deep web pada umumnya menggunakan aplikasi yang bernama Tor Browser. Tor Browser tidak hanya digunakan untuk membuat penggunanya menjadi anonim, tetapi tujuan utama dari Tor Browser adalah mengakses website yang miliki domain ".onion"

Dark Web

Dari kebanyakan sumber, dark web adalah kumpulan website atau yang kebanyakan mengarah pada forum-forum yang terdapat informasi, berita, atau apapun yang dapat dikatakan ilegal dan tidak pantas untuk konsumsi publik.

Bahaya Mengakses Deep Web dan Dark Web:  Terkena Jailbait

Source; trademarkia.com

Terkena Jailbait.

Sebelum membahas soal bahaya Jailbait di Deep Web alangkah baiknya kita mengetahui definisi lebih jauh dari istilah tersebut. Jailbait sendiri adalah suatu istilah yang merujuk kepada jebakan untuk berbuat tindakan seksual kepada seseorang di bawah umur yang memiliki penampilan layaknya orang dewasa. Di ranah Deep Web, hal tersebut cukup sering terjadi dan tak sedikit orang yang terjebak dengan Jailbait yang mana tentu saja sesuai namanya, umpan tersebut akan membawa mereka ke penjara karena tindakan asusila terhadap orang di bawah umur karena disana pun praktek penjualan anak di bawah umur untuk kebutuhan seksual masih marak terjadi. 

Bahaya Mengakses Deep Web dan Dark Web: Informasi Pribadi Bisa Terkuak

Source: oketekno.com

Informasi terkuak saat mengakses deep web.

Bahaya mengakses deep web dan dark web satu ini hampir sama seperti ancaman virus, saat mengakses konten yang ada di Deep Web atau Dark Web, kita juga harus siap-siap menghadapi bahaya lain yang mengintai yaitu bocornya informasi pribadi kita oleh para peretas. Bahkan lebih mengerikannya lagi, para hacker atau peretas juga bisa melacak posisi kita berada. 

Maka dari itu banyak yang menyarankan jika kita berani mengakses Deep Web atau Dark Web, harus dibekali dengan apa yang dinamakan VPN atau Virtual Private Network sebagai perangkat lunak untuk mengelabui posisi IP atau Internet Protocol kita agar informasi pribadi kita tidak mudah dilacak atau terkuak.

Bahaya Mengakses Deep Web dan Dark Web: Terjebak Cyber Crime

Source: teiss.co.uk

Terjebak cyber crime.

Kejahatan siber atau Cyber Crime menjadi dampak terbesar bagi siapapun yang mengakses hingga mengelola situs-situs Deep Web maupu Dark Web. Karena masuk ke dalam sisi gelap dari internet, siapa saja yang memasukinya harus siap-siap menerima bahaya serta konsekuensinya jika tidak memiliki keamanan yang baik saat menjelajah sisi lebih kelam dari internet melalui Deep Web ataupun Dark Web.

Banyak bahaya yang mengacu kepada Cyber Crime yang berkaitan dengan Deep Web dan Dark Web, beberapa diantaranya adalah praktek transaksi narkoba dan obat-obatan ilegal terlarang online serta penjualan manusia (human trafficking). Maka dari itu berhati-hati jika tidak ingin terjebak dalam ranah kriminal dari Deep Web maupun Dark Web, meskipun kita hanya sekedar iseng mengakses. Tanpa adanya keamanan internet yang kuat serta mental baja, siap-siap untuk menerima bahaya dan konsekuensi mengakses Deep Web dan Dark Web.

Untuk mengatur web dengan mode autentikasi user pada web server berbasis Apache, langkah-langkahnya adalah sebagai berikut:

Buka terminal atau console pada server.

Pastikan modul auth_basic dan modul authn_file sudah diaktifkan pada Apache dengan menjalankan perintah berikut:

sudo a2enmod auth_basic

sudo a2enmod authn_file

Buat file .htpasswd yang berisi daftar user dan password yang akan diizinkan untuk mengakses web. Gunakan perintah berikut untuk membuat file tersebut:

sudo htpasswd -c /etc/apache2/.htpasswd username

Ganti username dengan nama user yang ingin diizinkan untuk mengakses web.

Tambahkan konfigurasi autentikasi pada file .htaccess pada folder web yang ingin diatur. Jika file .htaccess belum ada, buat file tersebut dengan perintah:

sudo touch /var/www/html/.htaccess

Tambahkan konfigurasi autentikasi pada file tersebut dengan menambahkan kode berikut:

AuthType Basic

AuthName "Restricted Content"

AuthUserFile /etc/apache2/.htpasswd

Require valid-user

Konfigurasi ini akan membuat semua konten pada folder tersebut hanya dapat diakses oleh user yang terdaftar di file .htpasswd.

Restart Apache dengan perintah:

sudo systemctl restart apache2

Setelah konfigurasi selesai, saat user mencoba mengakses konten pada folder yang sudah diatur, mereka akan diminta untuk memasukkan username dan password yang sudah terdaftar pada file .htpasswd.

9. siapa penbuat deep web dan dark web


Deep Web (juga dikenal dengan nama Deepnet, Darknet, Invisible Web, Undernet atau Hidden Web) di Indonesia dapat disebut sebagai Web Dalam. Deep Web merupakan bagian dari World Wide Web tetapi tidak termasuk ke dalam internet yang dapat dicari dengan mudah, yaitu dengan menggunakan mesin pencari web yang menggunakan indeks mesin pencari web. Deep Web berbeda dengan Dark Internet, dimana komputer tidak dapat lagi dijangkau melalui Internet, atau dengan Darknet yang merupakan jaringan untuk menukarkan data, yang dapat digolongkan sebagai bagian kecil dari Deep Web.

Mike Bergman, pendiri BrightPlanet yang memberikan istilah tersebut,mengatakan bahwa mencari di Internet pada saat ini dapat dibandingkan dengan menjaring ikan di permukaan laut: akan mendapatkan banyak ikan yang tertangkap di jaring, tetapi informasi yang sangat dalam tidak akan tertangkap. Kebanyakan dari informasi yang terdapat di jaringan Internet, terkubur sangat dalam situs yang dinamis, dan mesin pencari web yang standar tidak dapat menemukannya. mesin pencari web yang tradisional tidak dapat "mendeteksi" atau mengambil data di dalam Deep Web. Halaman-halaman tersebut dianggap tidak ada hingga mereka tercipta secara dinamis sebagai hasil dari pencarian secara spesifik. Sejak tahun 2001, Deep Web telah dinyatakan memiliki ukuran yang lebih besar dari Web biasa.

CARA NONAKTIFKAN MODE GELAP DI WINDOWS

Hampir tidak bisa dipungkiri bahwa Anda akan menggunakan komputer Anda untuk waktu yang lama, terutama saat bekerja.

Untungnya, Windows 10 pengguna dapat mengurangi ketegangan mata yang potensial dengan mengaktifkan tema gelap, yang datang sebagai salah satu fitur pemutakhiran ulang tahun Windows 10. Skema warna level sistem ini dapat dibuat sedemikian rupa sehingga dapat menampilkan nada-nada gelap pada menu dan latar belakang, sehingga memudahkan anda melihatnya.

Tema gelap mengubah beberapa aplikasi seperti Microsoft Edge, kalender, Mail, Store, Pengaturan, Kalkulator, Alarm, dan jam untuk mode gelap juga.

Namun, itu tidak akan mengubah semua aspek sistem ke mode gelap. Misalnya, File Explorer mempertahankan skema warna default, sedangkan untuk aplikasi lain, Anda harus mengaktifkan mode gelap di pengaturan masing-masing.

Untuk menonaktifkan mode gelap atau tema gelap di Windows 10, lakukan langkah-langkah di bawah ini:

Langkah 1: Klik Mulai dan pilih ikon roda gigi untuk membuka Pengaturan.

Langkah 2: Klik Personalization.

Langkah 3: Pilih Warna lalu gulir ke bawah untuk memilih mode aplikasi default Anda.

Langkah 4: Klik Light

Jawaban:

Untuk mengaktifkan Dark Mode atau mode gelap di Windows 11, ada beberapa langkah yang dilakukan.

Pertama-tama, klik kanan pada tampilan utama (desktop) komputer Windows 11 Anda. Lalu, pilih menu "Personalize".

Di tampilan berikutnya, klik menu "Colors". Kemudian, klik menu drop-down di kolom "Choose your mode" dan klik pilihan "Dark".

Setelah dipilih, tampilan komputer Windows 11 Anda kini akan menjadi serba hitam dan tidak begitu silau di mata.

Selain menerapkan Dark Mode sepenuhnya dengan memilih menu "Dark", pengguna juga bisa memilih "Custom" untuk menerapkan mode yang berbeda di tiap tampilan Windows 11.

Ketika membuka aplikasi, misalnya, pengguna bisa menerapkan Light Mode supaya mudah dioperasikan, sedangkan untuk tampilan desktop Windows 11, seperti taskbar, pengguna bisa menerapkan Dark Mode agar lebih nyaman dilihat.

31. bagaimana cara mengatur web dengan mode autentikasi user pada web server berbasis apache​


Jawaban:

Untuk mengatur web dengan mode autentikasi pengguna (user authentication) pada server web Apache, dapat dilakukan dengan mengikuti langkah-langkah berikut:

Pastikan module autentikasi telah diaktifkan pada server Apache. Module autentikasi yang umum digunakan pada Apache adalah mod_auth_basic dan mod_auth_digest. Module ini dapat diaktifkan pada file konfigurasi httpd.conf atau melalui file konfigurasi terpisah yang disebut .htaccess.Buat file pengguna (user file) yang berisi daftar pengguna yang diizinkan mengakses web. File ini berisi informasi login pengguna dan kata sandi yang telah dienkripsi dengan menggunakan algoritma tertentu. File ini dapat dibuat dengan menggunakan utilitas htpasswd yang tersedia pada kebanyakan sistem operasi.Konfigurasikan file .htaccess pada direktori yang akan diatur autentikasinya. File .htaccess ini berisi aturan autentikasi yang digunakan oleh server Apache untuk mengatur akses ke direktori atau file tertentu. File .htaccess biasanya ditempatkan di direktori root (dokumen root) dari situs web, atau di direktori yang diinginkan untuk membatasi akses pengguna.Setelah itu, pada file .htaccess, buat aturan autentikasi dengan menggunakan directive "AuthType", "AuthName", "AuthUserFile", dan "Require". Direktif AuthType menentukan tipe autentikasi yang digunakan (misalnya, Basic atau Digest), AuthName menentukan nama realm (area) autentikasi, AuthUserFile menentukan lokasi file pengguna yang dibuat sebelumnya, dan Require menentukan aturan akses yang diperlukan untuk mengakses direktori atau file tertentu.

Contoh aturan autentikasi pada file .htaccess sebagai berikut:

AuthType Basic

AuthName "Restricted Area"

AuthUserFile /path/to/userfile

Require valid-user

Dalam contoh ini, tipe autentikasi yang digunakan adalah Basic, nama realm autentikasi adalah "Restricted Area", file pengguna terletak di /path/to/userfile, dan akses diizinkan untuk pengguna yang valid (terdaftar dalam file pengguna).

Setelah konfigurasi ini selesai, server Apache akan meminta pengguna untuk login dan memasukkan nama pengguna dan kata sandi sebelum dapat mengakses halaman web yang diatur autentikasinya.

Harap dicatat bahwa proses konfigurasi autentikasi pada server web Apache dapat berbeda-beda tergantung pada sistem operasi dan versi server Apache yang digunakan. Jadi, pastikan untuk mengikuti dokumentasi resmi dari Apache atau panduan administrasi sistem operasi yang digunakan.

Jawaban:

Untuk mengatur web dengan mode autentikasi user pada web server berbasis Apache, dapat mengikuti langkah-langkah berikut:

1. Aktifkan modul autentikasi Apache dengan menjalankan perintah berikut di terminal:

sudo a2enmod auth_basic

Setelah itu, restart Apache dengan menjalankan perintah:

sudo systemctl restart apache2

2. Buat file konfigurasi untuk autentikasi pada direktori yang akan diatur. lu dapat membuat file .htaccess pada direktori tersebut, atau membuat file konfigurasi pada file apache2.conf atau httpd.conf jika lu ingin menerapkan autentikasi pada seluruh direktori pada server.

3. Tambahkan konfigurasi autentikasi pada file .htaccess, apache2.conf, atau httpd.conf. Berikut adalah contoh konfigurasi untuk autentikasi dasar:

AuthType Basic

AuthName "Restricted Content"

AuthUserFile /path/to/.htpasswd

Require valid-user

4. Buat file .htpasswd yang berisi username dan password yang dienkripsi. lu dapat membuat file ini dengan menggunakan perintah htpasswd pada terminal. Contoh perintah:

sudo htpasswd -c /path/to/.htpasswd username

5. Restart Apache dengan menjalankan perintah:

sudo systemctl restart apache2

Setelah konfigurasi autentikasi selesai, pengguna yang mengakses direktori atau konten yang diatur akan diminta untuk memasukkan username dan password yang sesuai. Jika username dan password yang dimasukkan sesuai dengan yang ada di dalam file .htpasswd, maka pengguna dapat mengakses konten tersebut.

39. Mode pasar mode litmus


Jawaban:

1. Pasar Perdana (Primary Market)

Merupakan jenis pasar yang mana efek efek atau surat berharga diterbitkan langsung oleh emiten dan diperdagangkan untuk pertama kalinya ke masyarakat sebelum dicatatkan pada Bursa Efek Indonesia.

Untuk kegiatan pembelian efek pada pasar perdana memiliki batasan waktu sekitar 90 hari sejak Bapepam memberikan izin emisi resmi kepada masing masing emiten. Pasar perdana memiliki karakteristik yaitu jangka waktu pemesanan yang terbatas.

Emiten menjual saham kepada masyarakat melalui perantara penjamin emisi dan pembeli surat berharga tidak dipungut biaya transaksi. Untuk harga saham dipasar ini tidak berubah ubah (tetap). Hal tersebut dikarenakan sebuah perusahaan telah menentukan harga dan jumlah saham yang akan ditawarkan.

2. Pasar Sekunder (Secondary Market)

Secondary market merupakan pasar keuangan untuk memperdagangkan sekuritas atau surat berharga yang telah diterbitkan dalam penawaran umum perdana.

Pasar ini memberikan kesempatan kepada investor untuk memperjual-belikan efek yang sudah tercatat di Bursa Efek Indonesia, setelah dilakukannya penawaran melalui initial public offering (IPO). Setelah tercatat di Bursa Efek Indonesia, saham perusahaan dapat ditransaksikan secara bebas.

Proses transaksi penjualan dan pembelian efek pada pasar sekunder tidak lagi dilakukan investor dengan perusahaan. Akan tetapi antara investor satu dengan yang lain. Kelebihan pasar ini antara lain dapat menunjukan tolak ukur penilaian perusahaan tertentu, mobilisasi tabungan menjadi lebih mudah dan dana investor relatif aman.

3. Pasar Ketiga (Third Market)

Pasar ketiga merupakan tempat perdagangan saham yang ada di luar bursa atau biasa disebut over the counter market. Untuk proses transaksi dilakukan antara market dengan investor yang mana harganya sudah ditentukan oleh anggota bursa. Third market tidak memiliki pusat lokasi perdagangan (floor trading).

Namun, sistem operasi pasar ini berupa pemusatan informasi yang biasa disebut trading information. Untuk informasi yang diberikan berupa harga harga saham, jumlah transaksi dan keterangan lainnya tentang surat berharga. Di dalam sistem perdagangan ini, pialang bertindak dan berkedudukan sebagai pedagang efek maupun perantara.

4. Pasar Keempat (Forty Market)

Ada yang pernah mendengar apa itu fourth market? Pasar keempat adalah bentuk transaksi jual beli efek antara pemodal dari satu pemegang saham ke lainnya tanpa melalui perantara. Walaupun transaksi dalam pengalihan saham terjadi secara langsung antara pemodal satu dengan lainnya tetapi mekanisme kerjanya menghendaki pelaporan terhadap transaksi block sale.
